# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest | ✅ |
| Previous minor versions | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in LLMScope, please report it by emailing security@llmscope.dev.

### What to Include

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fix if you have one

### Response Timeline

- We will acknowledge receipt of your report within 24 hours
- We will investigate the issue and provide a status update within 72 hours
- We will work to fix the issue as quickly as possible
- We will notify you when the fix is released

## Security Best Practices

- Always use the latest version of LLMScope
- Keep your dependencies up to date
- Use strong authentication for any exposed APIs
- Regularly review your configuration for security best practices
